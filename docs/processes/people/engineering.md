# Engineering

This section covers the tools we use to write code, collaborate on development, and support our workflows with AI. Everything here helps us move faster, stay aligned, and build better.

## GitHub: code and collaboration

We use **GitHub** for version control, code reviews, and project development. You’ll be added to two GitHub organizations:

| GitHub Org            | Purpose                          | Access Details                                  |
|------------------------|----------------------------------|--------------------------------------------------|
| `Defi-Wonderland`     | Main project development         | Multiple teams based on your role and projects   |
| `Wonderland-Quests`   | Onboarding and quest reviews     | One team only, used for internal onboarding      |

### `Defi-Wonderland`

This is where the core work happens: production code, client projects, and public goods.

You’ll be added to multiple teams depending on:
- Your role (Dev, QA, Research, etc.)
- What you’re working on (or might be working on soon)

Use this space for:
- Submitting and reviewing pull requests
- Managing issues
- Writing and reading documentation

> ✅ Each repo may have its own contributing guidelines. Check the README before pushing code.

### `Wonderland-Quests`

This is our internal sandbox for onboarding and challenge reviews. You’ll only be part of one team here.

We use this org to:
- Practice our workflows
- Help new contributors get familiar with our standards
- Review submissions from people interested in joining Wonderland

Some of us were recruited through quests, we take them seriously.

## AI Tools: accelerating workflows

We use a few AI tools to support coding, writing, and research. You’ll get access to accounts and resources for all of them.

### ChatGPT: research and writing

We use the **Wonderland Team plan** on ChatGPT.

Use it for:
- Generating ideas or outlines
- Writing docs or improving drafts
- Analyzing or summarizing content

> ⚠️ Don’t paste sensitive client data, treat it like a public assistant.

### Cursor: AI-Powered IDE

**Cursor** is our coding environment, based on VS Code.

Use it for:
- Write and refactor code
- Understand new codebases
- Auto-complete functions
- Debug or generate tests

Try using Cursor when joining a new repo or working through tricky problems.

### OpenRouter: Multi-Model Access

**OpenRouter** gives access to a range of AI models from different providers, not just OpenAI.

Use it for:

- Access to non-OpenAI models (e.g., Claude, Gemini, Mixtral).
- Faster or cheaper API endpoints.
- To experiment with prompt variations across different models.

We use it because it gives the team flexibility to use the best model for each task. Also, it helps benchmark and test different LLM capabilities.

To access:
- Wonderland provides access to our OpenRouter API key and dashboard, so you can:
- Use it in your development environment (e.g., local apps, Cursor, notebooks).
- Run model comparisons or experiments across providers.

:::warning
You’ll receive a shared API key and dashboard access. Keep that key private — don’t push it to public repos.
:::
